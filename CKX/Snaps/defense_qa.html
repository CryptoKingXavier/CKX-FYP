What is the difference between voluntary and involuntary churn? Give a real Nigerian example of each.
Voluntary churn is customer-initiated — the subscriber actively decides to leave. Example: an MTN Nigeria customer frustrated with repeated call drops and high data costs decides to port to Airtel Nigeria, drawn by a competitor's cheaper data bundle promotion.

Involuntary churn is provider-initiated — the telecom company terminates the relationship. Involuntary churn can also result from fraud detection, network infrastructure discontinuation, or network mergers/acquisitions.

What is purposive sampling and why did you use it over random sampling?
Purposive sampling is a non-probability sampling technique where respondents are deliberately selected based on specific characteristics relevant to the research objectives. In this study (Chapter 3.8), it meant targeting customers who had actively interacted with telecom services over a defined period — particularly those exhibiting churn-related behaviors such as frequent service complaints, plan changes, or expressed switching intentions.

Purposive sampling ensures a more informative and analytically relevant sample for churn prediction modeling.

B5
What is min-max normalization, and which features required it? State the formula.
Min-max normalization (Chapter 3.4) rescales numerical features to a fixed range of [0, 1] using the formula:

x' = (x − x_min) / (x_max − x_min)

where x is the original value, x_min is the column minimum, x_max is the column maximum, and x' is the normalized value. The numerical features requiring scaling were: Age, Monthly Spending (NGN), Contract Length (months), Monthly Call Usage (minutes), and Monthly Data Usage (GB). Normalization was necessary because these features operate on vastly different scales — e.g., Age ranges from ~18 to 65+, while Monthly Spending can range from ₦1,000 to ₦20,001+. Without normalization, features with larger ranges would dominate distance-based algorithms like SVM.

The chi-square test is a statistical test that measures whether two categorical variables are independent or whether there is a significant association between them. D2
Explain how a Decision Tree works and how it makes a churn prediction.
A Decision Tree (Chapter 2.7) is a supervised learning algorithm that recursively partitions the feature space by asking a sequence of yes/no questions about feature values. Starting at the root node, the algorithm selects the feature that best separates churners from non-churners (based on criteria such as Gini impurity or information gain). Each split creates two or more branches representing different value ranges or categories. This continues down the tree until reaching leaf nodes, which output the final prediction (Churn = Yes or No).

For example, a path might read: "Is Monthly Spending < ₦5,000?" → Yes → "Are there >3 complaints?" → Yes → Predict Churn = Yes. The tree is trained on the 80% training set and the learned rules are applied to the test set. Its main advantage is interpretability — you can trace exactly why a customer was flagged as a churner.

D3
What is the difference between a Decision Tree and a Random Forest?
A Decision Tree is a single model that grows one tree from the training data. While interpretable, it tends to overfit — memorizing training patterns that do not generalize to new data — especially when the tree is allowed to grow deep.

A Random Forest is an ensemble method that builds many decision trees (typically hundreds), each trained on a random subset of the training data (bagging) and a random subset of features. Final predictions are made by majority vote across all trees. This diversity reduces variance and overfitting significantly. In this study, RF achieved 64% accuracy versus DT's 54%, directly demonstrating the benefit of the ensemble approach. As Breiman (2001) argues, combining multiple weak learners creates a strong, generalized predictor.

D4
What is XGBoost and how does it differ from Random Forest?
XGBoost (Extreme Gradient Boosting) is an ensemble method that builds trees sequentially — each new tree is trained to correct the prediction errors of the previous one. This is called boosting, and it uses gradient descent to minimize a loss function at each step. XGBoost also incorporates regularization (L1 and L2 penalties) to prevent overfitting, which makes it more robust than standard gradient boosting.

The key distinction from Random Forest is the training paradigm: RF uses bagging (parallel, independent trees, then aggregate), while XGBoost uses boosting (sequential, error-correcting trees). In practice, XGBoost often achieves higher accuracy on structured tabular data — it won many Kaggle competitions. In this study it achieved 61% accuracy with the highest precision of the four models (57%), indicating it was more selective (fewer false positives) but less sensitive than RF and SVM.

D5
How does a Support Vector Machine (SVM) classify customers as churn or no-churn?
SVM (Chapter 2.6.1) is a supervised learning algorithm that finds the optimal hyperplane — the decision boundary that maximally separates the two classes (Churn = Yes vs. No) in multi-dimensional feature space. The boundary is positioned to maximize the margin, which is the distance between the hyperplane and the nearest data points from each class (called support vectors).

For non-linearly separable data (which is typical in real-world churn datasets), SVM uses kernel functions (e.g., Radial Basis Function — RBF) to transform the feature space into a higher dimension where a linear separator can be found. SVM is particularly suited to high-dimensional spaces and datasets where the number of features is large relative to records. It achieved the highest accuracy (65%) in this study.

E1
Define accuracy, precision, recall, and F1-score. Provide the formula for each.
Key
From Chapter 3.7 and 2.10:

Accuracy = (TP + TN) / (TP + TN + FP + FN) — The proportion of all predictions that were correct. Useful overall but misleading on imbalanced datasets.

Precision = TP / (TP + FP) — Of all customers predicted to churn, how many actually churned? High precision means fewer false alarms.

Recall (Sensitivity) = TP / (TP + FN) — Of all customers who actually churned, how many did the model catch? High recall means fewer missed churners. This is the most critical metric for churn prevention.

F1-Score = 2 × (Precision × Recall) / (Precision + Recall) — The harmonic mean of precision and recall. Provides a single balanced metric, especially valuable when classes are imbalanced.

E3
What is a confusion matrix? Explain TP, TN, FP, FN in the context of churn prediction.
A confusion matrix is a 2×2 table that summarizes a classification model's predictions against actual outcomes (Figures 4.3.1–4.3.4). In churn prediction context:

True Positive (TP): Model predicted "will churn" and customer actually churned. Best outcome — enables proactive retention intervention.

True Negative (TN): Model predicted "will not churn" and customer stayed. Correct and efficient — no wasted retention spend.

False Positive (FP): Model predicted "will churn" but customer was not going to churn. A false alarm — wastes retention resources (e.g., offering discounts to loyal customers unnecessarily).

False Negative (FN): Model predicted "will not churn" but customer churned anyway. The most damaging error — a churner slips through undetected, representing lost revenue with no intervention attempt.

E4
Why is recall especially important in churn prediction?
In churn prediction, a False Negative (missing an actual churner) is far more costly than a False Positive (incorrectly flagging a loyal customer). When a churner is missed, the company loses that customer's entire future revenue with no opportunity to intervene. By contrast, offering an unnecessary retention incentive to a non-churner (false positive) typically costs the company far less — perhaps a discounted bundle or a loyalty reward.

This asymmetry of error costs means maximizing recall (catching as many true churners as possible) is the primary optimization goal for a real-world churn prediction system. SVM achieved the highest recall in this study at 65%, though this is caveated by its failure to correctly classify any individual churner in the churn class specifically (due to class imbalance effects on its decision boundary).

SVM achieved 65% accuracy but only 43% precision. How do you reconcile these two results?

F2
Did you apply any technique to address class imbalance? If not, what would you have done?
No class imbalance mitigation technique was explicitly applied in this study. The class distribution in the test set (903 non-churners vs. 480 churners) was used as-is.

Had this been addressed, the recommended approaches would include:

SMOTE (Synthetic Minority Oversampling Technique): Generates synthetic samples of the minority class (churners) by interpolating between existing minority samples — increasing their representation without simple duplication.

Class-weight balancing: Setting class_weight='balanced' in scikit-learn's SVM and RF implementations, which automatically adjusts loss function penalties to give the minority class proportionally higher weight during training.

Undersampling: Randomly reducing the majority class (non-churners) to match the minority class size — simpler but risks losing information.

These are flagged as priority improvements for future work (Chapter 5.3).

You defined ROC-AUC in Chapter 2 but never reported it in Chapter 4. Why?
Chapter 2.10 provides a thorough definition of ROC-AUC, but it does not appear in the Chapter 4 evaluation tables — this is an acknowledged gap in the study's evaluation completeness.

ROC-AUC (Receiver Operating Characteristic — Area Under Curve) is particularly valuable for imbalanced classification problems precisely because it evaluates model performance at all possible decision thresholds, not just the default 0.5. It provides a single score from 0 to 1 (0.5 = random guessing, 1.0 = perfect discrimination) that is independent of class distribution — making it far more reliable than accuracy for this dataset.

Had ROC-AUC been computed, it would very likely show SVM performing close to 0.5 (random-level discrimination between churn classes), while XGBoost and RF would score higher. This would fundamentally change the model ranking and recommendation. Future work should include ROC-AUC as a primary evaluation metric.

F5
What is overfitting, and how did you guard against it in your models?
Overfitting occurs when a model learns the training data too precisely — including noise and idiosyncrasies — and consequently performs poorly on new, unseen data. It manifests as a large gap between training accuracy and test accuracy.

Safeguards employed in this study include:

Random Forest: Bagging naturally reduces overfitting by introducing randomness in tree construction and averaging over diverse learners.
XGBoost: Includes built-in L1/L2 regularization terms in its objective function, penalizing model complexity.
Train-test split: Using a dedicated 20% holdout set that the model never sees during training provides an honest overfitting check.
Decision Tree: No explicit depth limiting is mentioned — this may explain its lowest performance (54%), as an unlimited DT can grow to perfectly fit training data while generalizing poorly.

Explicit learning curves comparing training vs. validation accuracy were not reported in the study, which would have been the most direct overfitting diagnostic.

F6
What is Cronbach's Alpha and what does a score of 0.85 mean for your survey?
Cronbach's Alpha is a coefficient of internal consistency — it measures how reliably a set of survey items collectively measure the same underlying construct. Values range from 0 to 1, with widely accepted benchmarks: ≥0.90 = excellent, ≥0.80 = good, ≥0.70 = acceptable, <0.60 = poor.

The pilot test in Chapter 3.9 (n=30 respondents) yielded α = 0.85, which falls in the "good" reliability category. This means the questionnaire items are consistently measuring the same churn-related constructs — survey responses were not random or contradictory. High internal consistency gives confidence that the instrument captured genuine customer sentiments about service quality, cost sensitivity, and switching intention rather than measurement noise.

Note that Cronbach's Alpha measures consistency but not validity — a highly consistent survey can still fail to measure the right constructs. Content validity (expert review) was also applied (Chapter 3.9) to address this.

F8
Why did the Decision Tree have only 54% accuracy — barely above random chance?
The Decision Tree's weak performance (54% accuracy) can be attributed to several causes:

1. Overfitting: Without depth constraints, a single DT can memorize the training data perfectly but fail to generalize. Its test performance nearly matching chance (50%) suggests poor generalization.

2. Class imbalance sensitivity: Unlike RF which averages across many trees, a single DT's split decisions can be dominated by the majority class (non-churners), misclassifying many churners.

3. High variance: Decision Trees are high-variance learners — small changes in training data produce very different trees. With a dataset of ~1,100 training records, this variance is not sufficiently suppressed.

Interestingly, DT had the most balanced class-level performance of all models (both classes had ~0.55 precision and ~0.54 recall), suggesting it was the only model making genuine predictions on both classes — albeit with low accuracy. This makes DT arguably more honest as a baseline than SVM despite the lower overall score.

What are the ethical implications of deploying a churn prediction model that labels customers as high-risk? Could this lead to discriminatory practices?
This is a genuinely important concern. Several ethical risks exist in production deployment:

Discriminatory profiling: If the model associates demographic features (age, occupation, education) with high churn risk, it could lead to differential treatment — e.g., withholding premium offers from low-income customers flagged as likely churners while offering them to higher-income customers. This could constitute algorithmic discrimination.

Manipulative retention practices: Targeting only at-risk customers with attractive offers while loyal customers receive no benefit creates perceived unfairness and may incentivize loyal customers to self-identify as churners to access better deals.

Privacy and consent: Customers must be informed that their behavioral data is being used for predictive modeling — a requirement under both GDPR and NDPR.

Fairness audits: Any deployed model should be regularly tested for disparate impact across demographic groups.

Chapter 2.11 touches on privacy and data protection but does not engage with algorithmic fairness — a gap that future work and any deployment plan must address.
